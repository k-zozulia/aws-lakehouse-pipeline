# S3 Data Lake Layout

This document describes the S3 bucket structure created for the AWS Lakehouse Pipeline homework.

## Bucket Details

- **Bucket name**: `karina-ecommerce-lakehouse`
- **Region**: eu-central-1 (Europe (Frankfurt)) — chosen for lower latency from Ukraine
- **Creation date**: February 2026
- **Purpose**: Centralized data lake for raw, curated, quarantined data and generated reports

## Folder Structure
s3://karina-ecommerce-lakehouse/
└── ecommerce/
├── raw/
│   └── orders/
│       └── orders_20260101.csv     ← raw input data from previous homework (CSV, ~530 KB)
├── curated/
│   └── orders/                     ← will contain Parquet files after Glue ETL job
│                                       (partitioned by order_date)
├── quarantine/
│   └── orders/                     ← rejected/invalid rows with rejection_reason column
└── reports/                        ← daily Markdown reports generated by Bedrock
